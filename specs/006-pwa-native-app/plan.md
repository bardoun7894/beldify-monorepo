# 006 — Beldify PWA: "Like a Real App"

**Goal:** Turn the Next.js storefront into a full installable PWA with offline support and
native Web Push notifications (NO Firebase). Laravel is the push *sender*, not a second app.

## Verified starting state (2026-06-06, live www.beldify.com)
- `manifest.json` served but basic (theme `#2563eb`, 0 shortcuts, no share_target).
- `/sw.js` → **404** — no service worker exists. No offline, no caching.
- `/firebase-messaging-sw.js` → **404** — push currently broken.
- Browser `PushManager` supported → native Web Push (VAPID) works without Firebase.
- Install-prompt UI already built: `EnhancedPWAContext`, `InstallPrompt`, `InstallBanner`,
  `ModernInstallPrompt`, mounted via `PWAProviderWrapper` in `layout.tsx`.
- Firebase footprint = only 3 files: `hooks/useFCM.ts`, `utils/fcm.ts`, `lib/firebase.ts`.
- Backend notification classes ALREADY exist (NewMessage, NewCommunityPost, NewResponse,
  CustomOrderCreated, CustomOrderStatus, OrderPlaced) — only need a WebPush channel.

## Architecture
- ONE installable PWA = `beldify-frontend` (Next.js). Customers/sellers install this.
- `beldify-backend` (Laravel) stores push subscriptions + sends Web Push via VAPID.
- Filament admin PWA = OUT of scope (separate audience, later).

## Library decision
- Drop unmaintained `next-pwa@5.6.0`. Use **`@serwist/next`** (Workbox-based, App-Router native).
- Wrap order: `withBundleAnalyzer(withSerwist(nextConfig))`.
- **Must wire into BOTH `next.config.js` (dev) AND `next.config.prod.js` (prod build copies this).**

## Push triggers (all chosen by user)
new chat message · reply on your Souk post · new order / buyer interest ·
order status update · new Open Souk request (custom order).

## Work packets

### Frontend (frontend-engineer)
1. Add `@serwist/next`, remove `next-pwa`. Wire serwist into next.config.js + next.config.prod.js.
2. Custom service worker (`app/sw.ts`): Serwist precache + runtime caching
   (app shell, images=SWR, API=SWR for offline browsing/cart), offline fallback page.
3. SW `push` + `notificationclick` handlers (native Web Push — title/body/icon/url payload).
4. Web Push subscribe flow: permission prompt → `PushManager.subscribe({ applicationServerKey: VAPID_PUBLIC })`
   → POST subscription JSON to Laravel. Unsubscribe on logout. NEXT_PUBLIC_VAPID_PUBLIC_KEY env.
5. Manifest polish: theme `#6366f1` (Atlas token, match layout), add `id`, `scope`,
   `shortcuts` (Souk, Messages, Cart, Orders), `share_target`, iOS splash + apple-touch-icons.
6. **Remove Firebase entirely**: delete useFCM/fcm.ts/lib/firebase, drop firebase deps,
   ensure RealtimeChatContext uses soketi/Pusher only. Stop /firebase-messaging-sw.js registration.
7. Consolidate the 3 install components → keep the one rendered by PWAProviderWrapper, delete dead.
8. Notification settings toggle in account/settings.

### Backend (backend-engineer)
1. Composer: `laravel-notification-channels/webpush` (minishlink/web-push under the hood).
2. VAPID keys → env (`VAPID_PUBLIC_KEY`, `VAPID_PRIVATE_KEY`, `VAPID_SUBJECT`). Generate via artisan.
   NOTE: 11.5k-line .env breaks phpdotenv → new env vars need container recreate (see KB).
3. Migration: `push_subscriptions` table (HasPushSubscriptions trait on User).
4. Endpoints: `POST /api/push/subscribe`, `DELETE /api/push/unsubscribe` (Sanctum).
5. Add `webpush` to `via()` + `toWebPush()` on the 6 existing notification classes above.
6. Remove any server-side FCM/kreait code if present.

### DevOps (backend-engineer role)
1. Prod MUST deploy via `docker-compose.prod.yml` (real `next build`) — SW only generated by build.
2. Serve `/sw.js` with `Cache-Control: no-cache` (CF pins stale workers otherwise).
3. Set VAPID env on prod (frontend public key baked at build; backend keys at runtime → recreate).

### QA (qa-engineer)
1. DevTools → exactly one SW active at scope `/`, no firebase-messaging-sw 404.
2. Offline: reload a visited product + cart with network off → still renders.
3. End-to-end push: subscribe → trigger each event → notification delivered + click deep-links.
4. Install on Android (real standalone) + iOS add-to-home-screen + splash.
