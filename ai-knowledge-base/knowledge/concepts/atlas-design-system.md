# Beldify Atlas Design System

## Overview
Beldify Atlas is the canonical design system for the Beldify frontend. It defines palette, typography, shape, motion, and component patterns for an editorial Moroccan-marketplace aesthetic. It is codified in `DESIGN.md` at the project root and used by every `/frontend-design`, `design-check`, `normalize`, `polish`, and `audit` skill pass.

## Key Points
- **Primary palette**: Atlas Indigo `#0E5C9B` (CTAs, headers, footer) · Saffron Amber `#C9A14A` (accents, badges, discount pills) · Marrakech Red `#9C2A2A` · Sand `#F7F3EE` (backgrounds)
- **Typography**: Playfair Display (display/H1) + Inter (body) — Playfair applied to editorial section titles, hero headlines, brand wordmark
- **Shape**: 12px border radius across components
- **Stitch project**: ID `2822550655708305761` ("Beldify — Moroccan Marketplace"), design-system ID `assets/1998950717374000813`
- **Applied across**: all 16 customer-facing pages — homepage, browse, PDP, cart, checkout, auth, content, shared Navbar + Footer
- **Source of truth**: `DESIGN.md` at the Beldify monorepo root; must be read before any frontend design work

## Details
The Atlas design system was created on 2026-05-14 using the Stitch MCP to generate an editorial Moroccan-marketplace aesthetic. It was codified in `DESIGN.md` so that all future skill passes (frontend-design → polish → audit) derive decisions from a single authoritative file rather than re-inventing the aesthetic on each pass.

The palette was iterated: an early pass tried purple gradients on white (rejected as generic AI aesthetics), then amber + indigo was confirmed by the user. The Stitch project was created with the name "Beldify — Moroccan Marketplace" and design-system "Beldify Atlas."

### Admin surface extension (2026-05-21)
Atlas tokens were extended to the Laravel admin backend on 2026-05-21. The shared chrome (sidebar, topbar, master layout) was refactored from Bootstrap 4/PixInvent to Atlas Tailwind-only with EN+AR translation keys. The dashboard was given Atlas KPI tiles (4-up grid with inline SVG sparklines via `atlasSparkPoints` PHP helper) and Chart.js revenue/orders series.

**Admin dashboard palette:**
- KPI tile backgrounds: amber-50 (revenue), indigo-50 (orders), rose-50 (issues), teal-50 (community) — Note: only the amber and indigo variants are strictly on-palette; rose and teal were used for visual differentiation across the 4 tiles
- Sidebar: indigo-950 background, amber accent on active item
- Top bar: flat indigo-900 (replacing the legacy purple gradient)
- Badge tokens: `updated` = indigo/amber chip, `pending` = amber chip, `new` = rose chip

### FeaturedSections off-palette fix (2026-05-21)
`components/home/FeaturedSections.tsx` (Next.js frontend) contained special-offer cards using `from-indigo-600 to-purple-600` and `from-pink-600 to-red-600` gradients — explicitly prohibited by DESIGN.md §2 ("Do NOT use: Purple gradients on white"). These were re-tokenized to Atlas Indigo-900 + Sand amber-50 with the canonical radial overlay from DESIGN.md §6.4. An Atlas empty state was added for when Best Sellers / New Arrivals API returns 0 items: amber-50/60 card + Sparkles icon + Playfair "The atelier is restocking" + "Post a brief" CTA.
