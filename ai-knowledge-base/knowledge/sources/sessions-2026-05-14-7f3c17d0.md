# Session 2026-05-14: Atlas frontend deploy, seeder fix, PWA banner suppression

## Summary
A long working session focused on stabilizing the Beldify production deploy after the Atlas (Indigo + Saffron Amber) design system landed via Stitch. The session covered four discrete fixes (Firebase warnings, missing static assets, PWA manifest, AccountSubControlSeeder FK violations) and confirmed the Atlas homepage rendered correctly on all 16 customer pages with 200 responses. A BuildKit `npm install` stall (no active subprocess for 13+ minutes) was resolved by killing the stuck build and starting the container from the existing fresh image — the bind-mount delivered source-code changes on container start, making the rebuild unnecessary.

## Key points
- **Firebase warnings silenced** — replaced `console.warn` in `firebase.ts:28` and `useFCM.ts:44` with silent assignment to `messagingDisabledReason`. Push notifications still require `NEXT_PUBLIC_FIREBASE_API_KEY`, `_PROJECT_ID`, `_AUTH_DOMAIN`, `_STORAGE_BUCKET`, `_MESSAGING_SENDER_ID`, `_APP_ID` and `_VAPID_KEY` env vars to actually work.
- **Static asset placeholders generated** — `hero-bg-1.jpg` / `hero-bg-2.jpg` (1920×1080 indigo gradients), `manifest-icon-192.maskable.png`, `manifest-icon-512.maskable.png`, `apple-icon-180.png`, `favicon-16x16/32x32.png`, plus PWA `manifest.json` with Beldify branding. All five assets return 200 from production.
- **AccountSubControlSeeder fix** — seeder was failing on NOT NULL `user_id`, `company_id`, `branch_id` columns. Patched to resolve the super-admin user, first company, first branch and inject the IDs into every row before insert. 46 rows seeded successfully.
- **Atlas homepage went live** — Stitch-designed homepage with "The souk" 8-category grid (editorial Unsplash photos, Playfair Display category names overlaid), indigo "Want it tailored to you?" bespoke strip with 3-step explainer and amber "Start a tailoring order" CTA, Mega Offers section, indigo footer. All 16 customer pages confirmed 200.
- **PWA install banner suppression** — required multiple iterations because surgical scroll-trigger fixes leaked. Final fix hard-disabled the modal at the component level in `EnhancedPWAContext.tsx`. Three trigger paths identified: first-paint auto-show, scroll-position-above-60%, and returning-visitor.
- **BuildKit `npm install` stall** — full `--no-cache` rebuild stuck for 13+ minutes with no active subprocess (`Sl` sleeping state). Daemon log showed `layer sha256… was not found (corruption?)` errors. Killed and brought up container from existing 2-hour-old image — functionally identical to clean rebuild because all source code arrives via bind-mount on container start. Saved 13 minutes of wasted build time.
- **Cloudflare edge cache** — confirmed still pinning `/_next/static/chunks/app/layout.js` (Navbar + Footer chunk) due to `Cache-Control: immutable` header. Manual CF dashboard purge required after every deploy until permanent next.config.js header override lands.

## Quotes
> The fresh `--no-cache` rebuild stuck on `npm install` inside BuildKit for 13+ minutes with no progress (no active subprocesses, `Sl` state). I killed it and brought up the container from **the image that was already rebuilt cleanly with `--no-cache` 2 hours ago**. That image is fresh enough — all the recent source-code edits arrive via the bind-mount on container start, so the running app has every Stitch / Atlas change.

> Push notifications still won't *work* until you set `NEXT_PUBLIC_FIREBASE_API_KEY`, `_PROJECT_ID`, `_AUTH_DOMAIN`, `_STORAGE_BUCKET`, `_MESSAGING_SENDER_ID`, `_APP_ID` (and `_VAPID_KEY` for FCM) in the frontend's runtime env, then rebuild/restart.
