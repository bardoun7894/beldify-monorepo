## Assistant

Fixed and deployed. Refresh `/en/categories` (and `/en/stocks`) — the TypeError is gone.

**Root cause:** `optional($model)` returns `Illuminate\Support\Optional` (not null), but `LanguageService::resolveTranslation(?Model $model, …)` declares the parameter as `?Model` — strict-typed to **Model or null only**. The `Optional` proxy doesn't pass.

**Fix:** drop `optional()`. The helper already guards null internally (`if (is_null($model)) return ''`), so passing `$category->user` directly is safe and correct.

Other catalog tables already used the raw `$model->relation` pattern (e.g. `translateField($customer, 'customer_name')`) — only the categories `user` field and the stocks `category`/`user` fields had the wrapper. All three patched + redeployed.
